## ✅ СДЕЛАНО (в прототипе, дизайн)
- Онбординг «Создание игрока»: имя, дата рождения, пол (без преднастройки — обязательный явный выбор),
  рост, вес, цель, активность → расчёт норм; выбор «Быстрый старт» / «Тест пробуждения, 3 дня»
- Необязательный блок фото тела в онбординге (3-4 ракурса, можно пропустить → базовая фигурка);
  явная одежда по полу + чекбокс согласия (не медицинское приложение)
- Экран Система: окно квеста в стиле System, аватар крупный без лишних зазоров, профессия-бейдж
- 6 характеристик на русском: 💪Сила / ⚡Ловкость / ❤️Здоровье / 🧠Интеллект / 🕯️Дух Воли / 🏃Выносливость
- Уровень, EXP, ранги E→D→C→B→A→S→SSS, эффективный ранг с помесячным локом (см. ⚠ ниже),
  таймер дня, штраф за неактивность
- 6 категорий ежедневного квеста (Сила/Ловкость/Выносливость/Интеллект/Здоровье/Дух Воли),
  минимум 1 упражнение из каждой — валидируется реально, не просто текстом
- EXP за упражнение масштабируется от фактической цели/объёма (volumeXp), а не фиксировано;
  потолок ×5 от базовой цели — анти-абьюз
- Множитель XP за серию без пропусков (×1.0…×1.30 на 7/14/30/90/180/365 дней)
- Допы/активные игры дают половину XP, если основной квест не выполнен на 100%, и доплачивают
  разницу до полной ставки при выполнении квеста; активные игры равноценны, награда зависит от
  выбранной длительности, а не вида спорта
- Внезапные задания Системы: редко (~2 раза в неделю), час на выполнение, живой таймер, бонусный XP
- Профессии: разблокируются на 20/45/80 уровне по доминирующей характеристике (Воин→Берсерк→
  Монарх Разрушения и аналоги для остальных 5 статов)
- Питание: кольцо калорий, БЖУ с нормами и цветом, дневник (что съел + время)
- Вода/напитки (Water Time): тип + объём + коэф. усвоения + тап
- Прогресс: анализ состава тела (полный список метрик), 3 способа ввода (фото/скрин весов/руками)
- AI-стилизованная фигура тела (3D, прозрачный фон, без подписей на самом фото) — генерируется
  в фоне после анализа, с явным статусом готовности в модалке
- Замеры сантиметром (11 параметров) + график тренда по выбранной части тела
- График веса интерактивный: период + метрика, статистика, синхронизация с облаком (мерж по дате)
- Серия дней — «денёчки», достижения в отдельном «Зале достижений»: крупные флип-карточки,
  сортировка получено/не получено, сводка с прогресс-баром, описание+награда на обороте
- Рейтинг: общий («Топ разносторонних» — ограничен слабейшей характеристикой, специалист не
  доминирует) + отдельные топы по каждой из 6 характеристик
- Клик по игроку в рейтинге → профиль (статы, серия, ранг) → заявка в друзья/принять/чат
- Колокольчик уведомлений (заявки в друзья + непрочитанные сообщения) на иконке «Рейтинг»
- Рабочий чат между друзьями: реальные сообщения (таблица messages), опрос раз в 4с, вкладка
  «Чаты» со списком переписок и непрочитанными
- Настройки приватности (5 переключателей — UI готов, бэк применения отложен)
- Вход по коду — один профиль на любом устройстве/браузере без привязки к Telegram-ID
- Бонус на день рождения: реально начисляет +500 XP раз в год (была чисто декоративная, теперь
  работает на каждый день рождения, не только при регистрации)
- Шаги: педометр через DeviceMotion API; ручной ввод как fallback; авто-засчёт при 10 000
- Бот без сервера: периодические проверки (неактивность, заявки в друзья) — GitHub Actions cron,
  бесплатно, без Render/постоянного процесса. bot.py — опционально, только для /start

## 📝 ПРИМЕЧАНИЕ ПО verify_jwt:false
Все edge functions у этого проекта развёрнуты с `verify_jwt:false` — это сделано осознанно
(мини-апп без своей авторизации, доступ открыт всем по дизайну), но это значит, что любой,
кто знает URL функции, может её вызвать. RLS на таблицах — основная защита данных, не auth
на функциях. Если будет реальная валидация Telegram initData (см. 🔒 выше) — можно сузить.
